# Contributing to GDI Launcher

Thank you for your interest in improving GDI Launcher! We welcome contributions from everyone, whether you are fixing a bug, adding a new feature, or improving the documentation. 

To keep the development process smooth and efficient, please follow these guidelines.

## 🚀 Getting Started

1. **Fork the Repository**: Create your own copy of the project on GitHub.
2. **Create a Branch**: Use descriptive branch names. Examples:
   - `feature/add-dark-mode`
   - `fix/window-scaling`
3. **Make Your Changes**: Write clean, maintainable code.
4. **Commit Your Changes**: Please use conventional commit messages (e.g., `feat: ...`, `fix: ...`).
5. **Submit a Pull Request (PR)**: Clearly describe the changes you made and why they are necessary.

## 🛠 Development Guidelines

* **Tech Stack**: The project is built with **Python** and **PySide6**. 
* **Code Style**: We strive to follow PEP 8 standards. 
* **Testing**: Please ensure that your changes do not break existing functionality. Verify that the launcher still manages instances correctly before submitting.

## 🐞 Bug Reports

If you encounter a bug, please:
* Check the **[Issues](https://github.com/BlueAxolotLArseniy/GDI-Launcher/issues)** page to see if it has already been reported.
* Open a new issue if needed.
* Provide a clear description of the problem, steps to reproduce it, and any relevant logs or screenshots.

## 💡 Feature Requests

Have an idea to make GDI Launcher better?
* Open an **[Issue](https://github.com/BlueAxolotLArseniy/GDI-Launcher/issues)**.
* Start the title with `[Feature]`.
* Explain how your idea would benefit the Geometry Dash community.
